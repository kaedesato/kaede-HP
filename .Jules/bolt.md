## 2024-05-18 - [Svelte Template Loop Optimization]
**Learning:** In Svelte templates, using internal `{#if}` conditions within an `{#each}` block to artificially limit the display of items (e.g., `{#if index < 3}`) forces the framework to iterate over the *entire* array during the render cycle. For components like preview grids that might receive large arrays (like a gallery album), this results in O(n) rendering complexity for operations that should visually be O(1).
**Action:** Use `.slice(0, n)` directly on the array within the `{#each}` block expression (e.g., `{#each array.slice(0, 3) as item}`). This ensures Svelte only iterates and creates reactive bindings for the exact number of elements needed, improving performance and reducing unnecessary layout recalculations. Handle overflow elements with a separate `{#if array.length > n}` block outside the loop.
